# 🛠️ Audio Transcriber（音訊轉逐字稿）

> 把錄音檔丟給小龍蝦，自動幫你轉成繁體中文逐字稿，還會標記誰在說話！

## ✨ 這個技能可以做什麼？

- 🎙️ 把音訊檔案（錄音、會議、Podcast）轉成**繁體中文逐字稿**
- 👥 自動辨識並標記多位說話者（說話者 A、說話者 B……）
- 📁 支援本機檔案、遠端網址、data URI，來源超彈性
- 🎵 支援多種音訊格式：`.mp3`、`.wav`、`.ogg`、`.flac`、`.m4a`、`.aac`、`.webm`、`.wma`
- 🤝 轉好的逐字稿可以再搭配 `meeting-note-formatter` 技能，整理成結構化的會議記錄

## 📦 安裝方式

在 Telegram 對話中輸入：

```
/skills
```

輸入後會顯示可安裝的技能列表，選擇此技能即可完成安裝。

## ⚙️ 需要的設定

使用前需要設定以下 API Key：

| 名稱 | 說明 |
| --- | --- |
| `GEMINI_API_KEY` | Google Gemini API 金鑰，用來呼叫語音辨識服務 |

請到 [Google AI Studio](https://aistudio.google.com/) 自行申請。

## 💬 提示詞範例

以下是你可以直接複製貼上跟小龍蝦說的話：

```text
請把這段錄音轉成繁體中文逐字稿。
幫我轉錄這個會議音檔，並區分不同說話者。
把這個 Podcast 片段轉成文字，無法辨識的地方請標記出來。
請將這個 m4a 檔整理成逐字稿。
我有一段錄音的網址，幫我做語音轉文字。
```

## 📝 輸出範例

小龍蝦會回傳像這樣的繁體中文 Markdown 逐字稿：

```
說話者 A：大家好，我們今天來討論一下下週的活動安排。

說話者 B：好的，我先報告場地的部分。目前已經確認借到大禮堂了。

說話者 A：太好了！那時間是幾點開始？

說話者 B：下午兩點，預計進行兩個小時。

[無法辨識]

說話者 A：沒問題，那我們就這樣決定吧。
```

> 💡 聽不清楚的地方會標成 `[無法辨識]`，不會亂猜喔！

## ⚠️ 注意事項

- 📏 音訊檔案大小建議在 **20MB 以內**，太大的檔案可能會失敗
- 🗣️ 主要針對**中文語音**最佳化，其他語言也會盡量轉成繁體中文
- 🔇 轉錄品質會受到背景噪音、說話重疊、音訊清晰度影響
- 🌐 如果是遠端網址，需確認連結可以正常存取
